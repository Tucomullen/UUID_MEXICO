## 1. Visión General y Process IDs

Todas las convenciones de nombre deben:
- Diferenciar entre código SAP estándar y código custom.
- Permitir un enfoque consistente en todo el desarrollo custom.
- Los objetos SAP estándar NO deben modificarse ni borrarse. Si se requiere un cambio, copiar el objeto con el nombre custom correspondiente.

## 2. Paquetes (Packages)

- Longitud máxima: 30 caracteres.
- Deben comenzar con `Z`.
- Objetos temporales: clase `$TMP` (pueden borrarse, no se migran).

**Formato:**

```
Z + DEV + <Sistema/Cliente> + <ProcessID>
```

| Componente | Valor |
|---|---|
| User development | Z |
| Environment | DEV |
| System/Client | ED2/300 |
| Process ID | XX (ver Apéndice B) |

**Ejemplos:**
```
ZDEV_ED2300_FI     → Finance
ZDEV_ED2300_XA     → Cross Applications
ZDEV_ED2300_TS     → Data Migration Test / Throw away
```

> **Nota:** Para objetos en `$TMP`, usar iniciales del desarrollador en el nombre. Ejemplo: `SAPMZDJF` para módulo pool de John Freeman en vez de `SAPMZD01`.

## 3. ABAP Dictionary (DDIC)

Longitud máxima general: 30 caracteres (algunos objetos tienen límites menores).

### 3.1 Tabla Transparente (Transparent Table)

Longitud máxima: **16 caracteres**.

| Prefijo | Descripción |
|---|---|
| `ZTT_` | Tabla transparente custom |

**Formato:** `ZTT_<ProcessID>_<ObjectID>`

**Ejemplo:** `ZTT_FI_TAB01`

---

### 3.2 Tabla Pool (Pool Table)

Longitud máxima: **16 caracteres**.

**Formato:** `ZPT_<ProcessID>_<ObjectID>`

**Ejemplo:** `ZPT_FI_TXT01`

---

### 3.3 Estructuras (Structures)

Longitud máxima: **30 caracteres**.

**Formato:** `ZST_<ProcessID>_<ObjectID>`

**Ejemplo:** `ZST_FI_STRUCT01`

---

### 3.4 Tabla de Verificación (Check Table)

Longitud máxima: **16 caracteres**.

**Formato:** `ZCT_<ProcessID>_<ObjectID>`

**Ejemplo:** `ZCT_FI_COBL`

---

### 3.5 Tabla de Idioma/Texto (Language/Text Table)

Longitud máxima: **16 caracteres**.

**Formato:** `ZLT_<ProcessID>_<ObjectID>`

**Ejemplo:** `ZLT_FI_COBL`

---

### 3.6 Grupo de Tipos (Type Group)

Longitud máxima: **5 caracteres**.

**Formato:** `ZTG<ObjectID>`

**Ejemplo:** `ZTGIO` (Type Group para Internal Orders)

---

### 3.7 Tipo de Tabla (Table Type)

Longitud máxima: **30 caracteres**.

**Formato:** `ZTY_<ProcessID>_<ObjectID>`

**Ejemplo:** `ZTY_FI_MAILRECP`

---

### 3.8 Vista (View)

Longitud máxima: **16 caracteres**. Los primeros 7 caracteres deben ser únicos.

**Formato:** `ZVW_<ProcessID>_<ObjectID>`

**Ejemplo:** `ZVW_FI_MAIL`

---

### 3.9 Dominio (Domain)

Longitud máxima: **30 caracteres**.

**Formato:** `ZDO_<ObjectID>`

**Ejemplos:**
```
ZDO_MONTH
ZDO_BUKRS   (copia del dominio estándar BUKRS)
```

---

### 3.10 Elemento de Datos (Data Element)

Longitud máxima: **30 caracteres**. Cada elemento nuevo debe vincularse a un dominio nuevo (no reutilizar dominios SAP estándar).

**Formato:** `ZDE_<ObjectID>`

**Ejemplo:** `ZDE_MONTH`

---

### 3.11 Objeto de Bloqueo (Lock Object)

Longitud máxima: **16 caracteres**. Debe comenzar con `EZ_`.

**Formato:** `EZ_<ObjectID>`

**Ejemplo:** `EZ_EBAN_01`

---

### 3.12 Ayuda de Búsqueda (Search Help)

Longitud máxima: **30 caracteres**.

**Formato:** `ZSH_<ObjectID>`

**Ejemplo:** `ZSH_SEARCH`

## 4. Programas ABAP

### 4.1 Programa Ejecutable

#### Programas de Prueba / Temporales
- Solo para entorno de desarrollo, no se transportan.
- Deben comenzar con `Y`.
- Deben crearse como Local Objects (`$TMP`).

**Formato:** `Y_<NombreLibre>`

**Ejemplo:** `Y_TKCBDOC_DISP`

#### Programas Productivos
- Se transportan a Test y Producción.
- Deben comenzar con `Z`.

**Formato:** `Z<ProcessID><RICEFW>_<NombreDescriptivo>`

| Posición | Valor | Descripción |
|---|---|---|
| 1 | Z | Namespace cliente SAP |
| 2-3 | XX | Process ID (ver Apéndice B) |
| 4 | C | Tipo RICEFW (1 carácter, ver Apéndice A) |
| 5 | _ | Separador (guión bajo) |
| 6-40 | Texto libre | Nombre descriptivo |

**Ejemplo:** `ZFIR_YEARLY_OPENITEMS` (Yearly Open Items Financial Report)

---

### 4.2 Includes

| Tipo de Include | Convención de Nombre |
|---|---|
| Data Definition (Global Data) | `<NombrePrograma>_TOP` |
| Selection Screen | `<NombrePrograma>_SELyy` |
| Class Includes | `<NombrePrograma>_LCLyy` |
| Performs (Subroutinas) | `<NombrePrograma>_FRMyy` |

> `yy` = número corrido 00-99.

---

### 4.3 Variante de Reporte (Report Variant)

Longitud máxima: **14 caracteres**.

**Formato:** `ZRV_<ObjectID>`

**Ejemplo:** `ZRV_SUP_1`

---

### 4.4 Module Pool

Longitud máxima: **30 caracteres**. Debe comenzar con `SAPxZ` donde `x`:
- `U` = update modules
- `M` = screen modules
- `D` = dialog modules
- `F` = subroutine modules

**Formato:** `SAP<x>Z<ProcessID><NNN>`

**Ejemplo:** `SAPMZFI001`

#### Componentes del Module Pool

| Tipo de Componente | Convención |
|---|---|
| Data Definition (Global Data) | `Z<NombrePrograma>_TOP` |
| PBO Module | `Z<NombrePrograma>_PBOyy` |
| PAI Module | `Z<NombrePrograma>_PAIyy` |
| Performs (subroutinas) | `Z<NombrePrograma>_FRMyy` |
| POH Module | `Z<NombrePrograma>_POHyy` |
| POV Module | `Z<NombrePrograma>_POVyy` |
| OK-CODE | Código de función significativo y lo más corto posible |
| Menu painter | `M<screen_no>_<XXXX>` |
| Forms para OK-CODE | `Form_<OK_CODE>` |
| MODULE Naming | `MOD_<NombreSignificativo>_<Screen_Number>` |

> `yy` = número corrido 00-99.

#### Controles y Elementos de Pantalla del Module Pool

| Nombre | Tipo | Convención |
|---|---|---|
| Text Field | Elemento | `EL_TXT_<FieldName>` |
| Input/Output Field | Elemento | `EL_FLD_<FieldName>` |
| Check Box | Elemento | `EL_BOX_<FieldName>` |
| Radio Button | Elemento | `EL_RDB_<FieldName>` |
| Push Button | Elemento | `EL_PBN_<OK-CODE>` |
| Tab Strip | Control | `CN_TBS_<ScreenNumber>_nn` (nn: 00-99) |
| Box | Control | `CN_BOX_<ScreenNumber>_nn` |
| Sub screen Area | Control | `CN_SSC_<ScreenNumber>_nnnn` (nnnn: 9000-9990, incremento 10) |
| Table | Control | `CN_TBL_<InternalTableName>` |
| Custom Container | Control | `CN_<NombreSignificativo>` |
| Status Icon | Elemento | `EL_STA_<FieldName>` |

---

### 4.5 Dynpro (Screen)

Rango de números reservado para dynpros custom: **9000 - 9999** (incremento de 10).

La pantalla inicial de una transacción debe ser **9000**.

---

### 4.6 GUI Status

Recomendación: máximo **4 caracteres** (el sistema permite hasta 20).

**Ejemplo:** `STND`

---

### 4.7 Código de Transacción (Transaction Code)

Longitud máxima: **20 caracteres**. Debe comenzar con `Z`.

**Formato:** `Z<NNN>` (NNN: 001 a 999)

**Ejemplo:** `Z001`

## 5. ABAP Core Data Services (CDS)

### 5.1 CDS Data Definitions

Longitud: **35 caracteres**. Deben comenzar con `ZCDS_DDL_`.

**Formato:** `ZCDS_DDL_<ProcessID>_<ObjectID>`

**Ejemplo:** `ZCDS_DDL_BW_FINANCEDATES`

### 5.2 Resumen de otros objetos CDS

Los siguientes tipos de entidades CDS siguen las reglas generales del framework ABAP CDS de SAP. No tienen convención de nombre custom adicional más allá del prefijo `Z`:

- CDS Annotations
- CDS Type Definitions (Simple Types, Enumerated Types)
- CDS Function Definitions / Scalar Functions
- CDS View Entities
- CDS Projection Views
- CDS Table Functions
- CDS Hierarchies
- CDS Custom Entities
- CDS Abstract Entities
- CDS Tuning Objects
- CDS Access Control
- CDS System Entities

## 6. Objetos de Función

### 6.1 Grupo de Funciones (Function Group)

**Formato:** `ZFG_<ProcessID>_<ObjectID>`

**Ejemplo:** `ZFG_FI_DATES`

---

### 6.2 Módulo de Función (Function Module)

Longitud máxima: **30 caracteres**. Debe comenzar con `ZFM_`.

**Formato:** `ZFM_<ProcessID>_<ObjectID>`

**Ejemplo:** `ZFM_FI_CHECK_DATE`

## 7. BAPI / BOR

| Tipo de Objeto SAP | Longitud | Convención |
|---|---|---|
| Parámetros en function module | 20 máx. | Máx. 20 caracteres alfanuméricos |
| Naming methods in BOR | 30 | `Z<NombreBAPI>` — componentes identificados con CamelCase. Ej: `ExistenceCheck`, `GetDetail` |
| Modified SAP delivered methods | 30 | `Z<NombreOriginalSAP>` + sufijo numérico si hay cambios incompatibles |
| Non-SAP developed function module | 30 | `ZBAPI_<BusinessObjectName>_<MethodName>` |
| Modified SAP delivered function module | 30 | `ZBAPI_<BusinessObjectName>_<MethodName>` |

## 8. Clases ABAP

Longitud máxima: **30 caracteres**.

**Formato:** `ZCL_<Subtype>_<ProcessID>_<ObjectID>`

| Subtype | Descripción |
|---|---|
| IM | Class implementation |
| SI | Service Interface |

**Ejemplo:** `ZCL_IM_FI_VENDOR_IBAN_ADD`

## 11. Variables de Programa

Las variables ABAP pueden tener hasta **30 caracteres** para campos DATA y subrutinas. Máximo **8 caracteres** para SELECT-OPTIONS y PARAMETERS.

Usar guión bajo `_` como separador. Usar `LIKE` siempre que sea posible.

### Prefijos de Variables

| Tipo | Global | Local |
|---|---|---|
| Parameters | `P_` | `P_` |
| Context Variable | `K_` | `K_` |
| Variable | `GV_` | `LV_` |
| Constants | `GC_` | `LC_` |
| Select Options | `S_` | `S_` |
| Internal tables | `GT_` | `LT_` |
| TYPES | `GTY_` | `LTY_` |
| Class Objects | `GO_` | `LO_` |
| Structure / Work Areas | `GS_` | `LS_` |
| Type Table Declaration | `GTT_` | `LTT_` |
| Control | `CN_` | `CN_` |
| OLE Objects | `OLE_` | `OLE_` |
| Classes | `ZCL_` | `LCL_` |

## 12. Otros Objetos

### 12.1 Menú de Área (Area Menu)

Longitud máxima: **20 caracteres**. Sigue las mismas convenciones que los Transaction Codes.

**Formato:** `Z<ProcessID>_<ObjectID>`

**Ejemplo:** `ZFI_TABS`

---

### 12.2 Mensajes (Messages)

Clase de mensaje: 20 caracteres. Número: 3 dígitos (000-999).

**Formato de clase:** `Z<ProcessID><ObjectID>`

> Solo puede existir **una clase de mensajes** por Process ID. Solicitar rango de números al DV Team Lead.

---

### 12.3 Job de Background (Background Job)

Longitud máxima: **32 caracteres** (se usan 25).

**Formato:** `<NombrePrograma>_<P/T><FreqMode><C/N>_<NNN>`

| Posición | Valor | Descripción |
|---|---|---|
| 1-9 | Nombre programa | Nombre del programa |
| 10 | `_` | Separador |
| 11 | `P` / `T` | Production / Test |
| 12 | Frecuencia | Ver tabla abajo |
| 13 | `C` / `N` | Critical / Non-Critical |
| 14-21 | SAP Logon | Login del solicitante |
| 22 | `_` | Separador |
| 23-25 | NNN | Número secuencial |

**Modos de Frecuencia:**

| Código | Descripción |
|---|---|
| D | Daily |
| W | Weekly |
| M | Monthly |
| Q | Quarterly |
| H | Half-Yearly |
| A | Annually |
| R | On Request |

**Ejemplo:** `ZSUPPLIER01_PMC_001`

---

### 12.4 Batch Input Session

**Formato:** `<ProgramID_sinZ><FreqMode><CC>`

- `ProgramID_sinZ` = Nombre del programa sin el prefijo `Z` (9 chars)
- `FreqMode` = Q/M/W/D/H/A (Adhoc)
- `CC` = Contador incremental (01, 02, ...)

**Ejemplo:** `PP0213I01A01`

## 15. Business Workflow

### 15.1 Business Object

- **Object Type:** 10 caracteres. Si deriva de estándar SAP: `Z<TipoOriginal>`. Ej: estándar `BUS0024` → custom `ZBUS0024`.
- **Object Name:** 20 caracteres. Si deriva de estándar: `Z<NombreOriginal>`.

### 15.2 Workflow Template

- **Template ID:** Generado según prefijo en Workflow Customizing (ej. prefijo 900 → `WS90000001`).
- **Abreviación:** 12 caracteres. Copia de estándar: `Z<AbrOriginal>`. Ej: `RequestAppr` → `ZRequestAppr`.

### 15.3 Task Template

- **Task ID:** Generado según prefijo en Workflow Customizing.
- **Abreviación:** 12 caracteres. Si es copia: `Z<AbrOriginal>`.

### 15.4 Rule (Regla)

- **Rule No:** 8 dígitos numéricos.
- **Abreviación:** 12 caracteres. Si es copia: `Z<AbrSignificativa>`.

## 16. LSMW

### 16.1 Objetos LSMW

Longitud máxima de cada objeto: **15 caracteres**.

| Objeto LSMW | Convención | Ejemplo |
|---|---|---|
| Project | `<ACRONIMO_CLIENTE>_DM_PROJECT` | `DATA_MIGRATION` |
| Subproject | `<ProcessID>_<SubProcessID>` | `SCM_MM` |
| Object | `<Legacy>_<Action>` | `ORA_CREATE` |

### 16.2 Gestión de Ficheros LSMW

| Directorio | Ruta |
|---|---|
| Pre-Processed | `\<cliente>\conv\input\pre` |
| Formatted | `\<cliente>\conv\output\format` |
| Mapped | `\<cliente>\conv\output\map` |
| Archived | `\<cliente>\conv\output\arc` |
| Errors | `\<cliente>\conv\output\error` |
| Processed | `\<cliente>\conv\output\processed` |

### 16.3 Nombres de Fichero

| Posición | Valores | Descripción |
|---|---|---|
| 1-25 | Nombre significativo | Conversion ID |
| 26-29 | `.arc`, `.prs`, `.err`, `.map`, `.inp`, `.pre` | Extensión (Archived, Processed, Error, etc.) |

## 17. Manejo de Excepciones

### 17.1 Errores de Programa
- Notificar al System Admin con: código de retorno, clave usada para acceso a BD, ubicación en el programa.
- La acción de terminación depende del contexto de ejecución.

### 17.2 Captura de Excepciones del Sistema

```abap
" Usar CATCH/ENDCATCH en lugar de dejar dumpear el programa
CATCH EXCEPTION DBIF_DSQL2_SQL_ERROR.
  " Presentar mensaje detallado al usuario con:
  " - Severidad del problema
  " - Causa del problema
  " - Remedios necesarios
  " - Cómo reprocesar la tarea
```

Reglas:
- Usar una **Error Class** en el CATCH siempre que sea posible.
- Solo usar Error ID específico si el error está muy precisamente definido.
- Cada Error ID/Class en línea separada del CATCH.
- Chequear el código de retorno inmediatamente después del ENDCATCH.
